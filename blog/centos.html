UGO的含义

U 文件的拥有者或者创建者
G 在文件的所属组（默认是创建文件的用户的主组）里的用户
O 既不是文件创建者，也不是文件属组里的用户 称为其他人
a 表示所有用户，包含UGO

修改文件权限
chmod [选项] 模式 文档
常见选项
-R 递归更改目录和目录里文件的权限

通过字母形式更改文件权限

chmod o+w 需要改权限的文件或目录 给o增加w权限
chmod u=rws 需要改权限的文件或目录 给u增加所有权限
chmod u=rws,g=rw,o=— 需要改权限的文件或目录 给u所有权限 给g r+w权限 取消o的所有权限
chmod +x 需要改权限的文件或目录 给所有人加x权限
chmos -R 不只改目录权限，还可以更改目录里面所有文件的权限
通过数字形式更改文件权限

rw- r-x r– 用数字表示就是6 5 4
chmod 777 需要改权限的文件或目录 给所有人所有权限 444 所有人给r权限 等等
chmod 777 目录/* 目录下所有文件都改成777权限

计划任务
语法 crontab 选项
选项
-l 列出用户计划任务列表
-e 编辑计划任务列表
-u 指定用户名，不指定，默认当前用户
-r 删除指定用户的计划任务列表

编辑计划任务
计划任务语法格式，一行一个计划 分 时 日 月 周 需要执行的命令
比如 0 0 * * * reboot 表示每天0点0分执行重启操作
取值范围
分：059
时：032
日：131
月：112
周：0~7 0和7表示星期天
四个符号
*：表示取值范围中的每一个数字
-：连续区间表达式，比如1-7
/：表示每多少个 ，每十分钟一次 */10
,：表示多个取值，比如在1点，2点，6点执行， 1,2,6

设置某些用户不允许设置计划任务(黑名单)
配置文件位于 /etc/cron.deny 里面写用户名，一行一个

还有一个配置文件（白名单）
/etc/cron.allow 里面写用户名，一行一个

白名单优先级高于黑名单

让普通用户可以有权限执行reboot等，需要root用户才能执行的命令
可以用sudo 命令来进行权限设置 sudo的配置文件 /etc/sudoers 配置sudo需要使用visudo命令，打开后使用方法和vim一致
配置权限
root ALL(ALL) ALL
root 表示用户名
ALL 表示允许登录的主机（地址白名单）
（ALL）表示以谁的身份执行，ALL表示root身份
ALL 表示当前用户可以执行的命令，多个命令可以使用“,”分割

添加规则命令的时候不建议写直接形式命令，而是写命令的完整路径
查看命令路径可以用which命令查看
语法 which 命令指令名称
配置好了以后，普通用户要执行命令需要用sudo来执行
语法 sudo 需要执行的指令
比如授权test用户实行添加用户，修改用户密码的权限，不允许修改root密码
test ALL(ALL) /usr/sbin/useradd,/usr/bin/passwd [A-Za-z]*, !/usr/bin/passwd root
先允许test修改所有用户的密码 后面再禁止修改root的密码

普通用户查看自己的特殊权限使用 sudo -l 命令

exit 退出用户
groupadd admin 添加admin组 groupdel admin 删除admin组

以用户的角度
usermod -G admin 用户名 修改用户的附加组到admin
以组的角度
gpasswd -M 用户1,用户2,用户3 把这三个用户一起加入到admin组

给文件添加执行权限 chmod +x test.sh
执行 /root/test.sh

变量 先定义再使用
定义形式 class_name=”hello word” 变量后面的等号左右不能有空格
使用形式 echo $class_name
使用变量改写上面的例子内容
#!/bin/bash
hc=”hello word!”
echo $hc

关于单双引号 双引号能够识别变量，单引号不能识别变量，只会原样输出。双引号能够实现转义，单引号不能转义
注意：当在脚本中需要执行一些指令并且将执行的结果赋予给变量的时候需要使用反引号”`”(Esc键下面)

只读变量
语法 readonly 变量名
例如
#!/bin/bash
hc=”holle word!”
readonly hc

接收用户输入
语法 read -p 提示信息 变量名
例如 执行之后提示请输入要创建文件的路径，然后自动为用户创建文件，提示文件创建成功，并列出文件
#!/bin/bash
read -p ‘请输入要创建文件的路径：’ hc

18.2. 命令参数：

  • -1 强制scp命令使用协议ssh1
  • -2 强制scp命令使用协议ssh2
  • -4 强制scp命令只使用IPv4寻址
  • -6 强制scp命令只使用IPv6寻址
  • -B 使用批处理模式（传输过程中不询问传输口令或短语）
  • -C 允许压缩。（将-C标志传递给ssh，从而打开压缩功能）
  • -p 留原文件的修改时间，访问时间和访问权限。
  • -q 不显示传输进度条。
  • -r 递归复制整个目录。
  • -v 详细方式显示输出。scp和ssh(1)会显示出整个过程的调试信息。这些信息用于调试连接，验证和配置问题。
  • -c cipher 以cipher将数据传输进行加密，这个选项将直接传递给ssh。
  • -F ssh_config 指定一个替代的ssh配置文件，此参数直接传递给ssh。
  • -i identity_file 从指定文件中读取传输时使用的密钥文件，此参数直接传递给ssh。
  • -l limit 限定用户所能使用的带宽，以Kbit/s为单位。
  • -o ssh_option 如果习惯于使用ssh_config(5)中的参数传递方式，
  • -P port 注意是大写的P, port是指定数据传输用到的端口号
  • -S program 指定加密传输时所使用的程序。此程序必须能够理解ssh(1)的选项。
